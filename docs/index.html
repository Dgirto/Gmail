1. ¿Qué es y para qué sirve?1. What is it and what is it for?

El conector Gmail permite que el agente de Ruvic lea, busque y envíe correos desde un buzón de Google Workspace, sin que nadie tenga que iniciar sesión manualmente ni volver a autorizar accesos. Una vez configurado, puedes pedirle al agente cosas como:

  • “Revisa si llegaron correos nuevos de soporte y resúmelos.”
  • “Busca los correos de la última semana que mencionen ‘factura pendiente’.”
  • “Envía un correo al cliente confirmando que el ticket fue resuelto, con el reporte adjunto.”

Operaciones soportadas:

The Gmail connector lets the Ruvic agent read, search and send email from a Google Workspace mailbox, without anyone having to sign in manually or re-authorize access. Once configured, you can ask the agent things like:

  • “Check if new support emails arrived and summarize them.”
  • “Search last week's emails mentioning ‘pending invoice’.”
  • “Email the client confirming the ticket was resolved, with the report attached.”

Supported operations:

OperaciónOperation Qué haceWhat it does
mail.list Busca/lista correos usando filtros (sintaxis de búsqueda de Gmail)Searches/lists emails using filters (Gmail search syntax)
mail.read Lee el detalle y el cuerpo de un correoReads the detail and body of an email
mail.send Envía un correo de textoSends a text email
mail.send_with_attachment Envía un correo con un archivo adjunto (máx. 20 MB)Sends an email with a file attachment (max 20 MB)
Un solo login, no en cada ejecuciónOne login, not on every run El conector usa OAuth2 con refresh token: el dueño del buzón autoriza el acceso una sola vez (con un script local, fuera de Ruvic) y ese permiso queda vigente. El agente nunca vuelve a mostrar una pantalla de "iniciar sesión con Google". Funciona tanto con cuentas de Gmail personales como con Google Workspace. The connector uses OAuth2 with a refresh token: the mailbox owner authorizes access once (with a local script, outside Ruvic) and that grant stays valid. The agent never shows a "sign in with Google" screen again. It works with both personal Gmail accounts and Google Workspace.

2. Prerrequisitos2. Prerequisites

  • Una cuenta de Gmail (personal @gmail.com o de Google Workspace — ambas funcionan igual con este conector).
  • Acceso al Google Cloud Console con permisos para crear un proyecto y credenciales OAuth.
  • Python 3.10+ instalado localmente, solo para ejecutar una vez el script que obtiene el refresh token (no hace falta en el runtime de Ruvic, ya viene preinstalado ahí).
  • Acceso al buzón que se va a conectar, para iniciar sesión y autorizar el acceso.
  • A Gmail account (personal @gmail.com or Google Workspace — both work the same with this connector).
  • Access to Google Cloud Console with permissions to create a project and OAuth credentials.
  • Python 3.10+ installed locally, only to run the refresh-token script once (not needed in the Ruvic runtime, which already has it preinstalled).
  • Access to the mailbox being connected, to sign in and authorize access.

3. Configuración en el fabricante (paso a paso)3. Vendor-side setup (step by step)

Vas a crear credenciales OAuth en Google Cloud y luego autorizar el acceso al buzón una sola vez, con un script que se ejecuta en tu computadora (no en Ruvic). El resultado es un refresh token que se pega en el formulario del conector.

You will create OAuth credentials in Google Cloud and then authorize access to the mailbox once, with a script that runs on your computer (not in Ruvic). The result is a refresh token that you paste into the connector form.

  1. Crea (o selecciona) un proyecto en Google Cloud Console.

    Ve a console.cloud.google.com/projectcreate y crea un proyecto (o reutiliza uno existente).

    Create (or select) a project in Google Cloud Console.

    Go to console.cloud.google.com/projectcreate and create a project (or reuse an existing one).

  2. Habilita la Gmail API.

    En el menú APIs y servicios → Biblioteca, busca “Gmail API” y pulsa Habilitar.

    Enable the Gmail API.

    In APIs & Services → Library, search for “Gmail API” and click Enable.

  3. Configura la pantalla de consentimiento OAuth.

    En APIs y servicios → Pantalla de consentimiento de OAuth: tipo de usuario Externo (obligatorio si usas una cuenta @gmail.com personal), completa el nombre de la app y el correo de contacto, y en la sección de usuarios de prueba agrega el correo del buzón que vas a conectar. Mientras la app quede en estado Testing, esto evita el proceso de verificación de Google.

    Configure the OAuth consent screen.

    In APIs & Services → OAuth consent screen: user type External (required if using a personal @gmail.com account), fill in the app name and contact email, and under test users add the email of the mailbox you're connecting. While the app stays in Testing status, this skips Google's verification process.

  4. Crea el cliente OAuth (tipo "Aplicación de escritorio").

    En APIs y servicios → Credenciales → Crear credenciales → ID de cliente de OAuth. Tipo de aplicación: Aplicación de escritorio. Anota el Client ID y el Client Secret generados (o descarga el JSON).

    Create the OAuth client (type "Desktop app").

    In APIs & Services → Credentials → Create credentials → OAuth client ID. Application type: Desktop app. Note the generated Client ID and Client Secret (or download the JSON).

    [ Captura: pantalla de creación de cliente OAuth tipo "Aplicación de escritorio" — reemplazar por assets/img/gmail-oauth-client.png ] [ Screenshot: OAuth client creation screen, "Desktop app" type — replace with assets/img/gmail-oauth-client.png ]
  5. Obtén el refresh token con el script incluido.

    Desde el paquete del conector, en tu computadora:

    pip install google-auth-oauthlib
python get_refresh_token.py --client-id TU_CLIENT_ID --client-secret TU_CLIENT_SECRET

    Se abre el navegador: inicia sesión con el buzón que quieres conectar y acepta los permisos solicitados (leer y enviar correo). El script imprime client_id, client_secret y refresh_token — copia los tres.

    Get the refresh token with the included script.

    From the connector package, on your computer:

    pip install google-auth-oauthlib
python get_refresh_token.py --client-id YOUR_CLIENT_ID --client-secret YOUR_CLIENT_SECRET

    Your browser opens: sign in with the mailbox you want to connect and accept the requested permissions (read and send mail). The script prints client_id, client_secret and refresh_token — copy all three.

4. Permisos requeridos4. Required permissions

Principio de mínimo privilegio: el conector solo necesita estos dos scopes de Gmail durante la autorización (paso 5 de la sección 3). No agregues scopes de administración, de Drive ni de otras APIs de Workspace al script.

Least-privilege principle: the connector only needs these two Gmail scopes during authorization (step 5 of section 3). Do not add admin, Drive, or other Workspace API scopes to the script.

ScopeScope Para qué lo usa el conectorWhy the connector needs it
gmail.readonly Buscar y listar correos (mail.list), leer el detalle de un correo (mail.read)Search and list emails (mail.list), read an email's detail (mail.read)
gmail.send Enviar correos (mail.send, mail.send_with_attachment)Send emails (mail.send, mail.send_with_attachment)

El conector no solicita gmail.modify ni gmail.settings.*: no puede eliminar correos, cambiar etiquetas ni modificar la configuración del buzón.

The connector does not request gmail.modify or gmail.settings.*: it cannot delete emails, change labels, or modify mailbox settings.

6. Verificación6. Verification

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa al buzón soporte@tuempresa.com”. La tarjeta del conector cambia a estado Configurado.

    Successful connection test.

    The Test connection button should return: “Successful connection to mailbox soporte@tuempresa.com”. The connector card switches to Configured.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele:

    “Busca en Gmail los correos no leídos de los últimos 3 días y resúmelos.”

    El agente debe responder con correos reales del buzón configurado. Si lo hace, el conector quedó operativo de punta a punta.

    Minimal test automation.

    Open a new conversation with the agent and ask:

    “Search Gmail for unread emails from the last 3 days and summarize them.”

    The agent should reply with real emails from the configured mailbox. If it does, the connector is fully operational.

8. Preguntas frecuentes y notas de seguridad8. FAQ and security notes

¿El agente puede leer o enviar correos desde cualquier buzón?Can the agent read or send email from any mailbox?

No. Solo puede actuar como el buzón que autorizó el refresh token en el paso 5 de la sección 3. Para dar acceso a otro buzón, esa persona debe correr get_refresh_token.py con su propia sesión (y crear otra instancia del conector con ese token).

No. It can only act as the mailbox that authorized the refresh token in step 5 of section 3. To grant access to another mailbox, that person must run get_refresh_token.py with their own session (and create another connector instance with that token).

¿El agente puede borrar correos o cambiar la configuración del buzón?Can the agent delete emails or change mailbox settings?

No. El conector solo solicita los scopes gmail.readonly y gmail.send; Gmail rechaza cualquier operación fuera de esos alcances aunque el código lo intentara.

No. The connector only requests the gmail.readonly and gmail.send scopes; Gmail rejects any operation outside those scopes even if the code attempted it.

¿Dónde queda guardado el refresh token?Where is the refresh token stored?

Como secreto en el almacén de credenciales de la plataforma, asociado a tu usuario, igual que el client_secret. La interfaz nunca vuelve a mostrarlos; solo indica “definido”. Para cambiarlos, pega valores nuevos en el formulario; para conservarlos al editar otros campos, deja esos campos vacíos.

As a secret in the platform's credential store, tied to your user, same as the client_secret. The UI never displays them again; it only shows “set”. To change them, paste new values into the form; to keep them while editing other fields, leave those fields empty.

¿Expira el refresh token?Does the refresh token expire?

No, mientras la app OAuth de Google Cloud esté publicada ("In production") y el usuario no revoque el acceso manualmente. Si la app sigue en estado Testing, Google expira el refresh token a los 7 días sin uso — en ese caso hay que repetir el paso 5 de la sección 3.

No, as long as the Google Cloud OAuth app is published ("In production") and the user doesn't manually revoke access. If the app is still in Testing status, Google expires the refresh token after 7 days without use — in that case, repeat step 5 of section 3.

¿Qué hago si el refresh token se compromete?What if the refresh token is compromised?

  1. El dueño del buzón entra a myaccount.google.com/permissions y revoca el acceso de la app conectada.
  2. Corre de nuevo get_refresh_token.py para generar un token nuevo.
  3. Actualízalo en el formulario del conector.
  1. The mailbox owner goes to myaccount.google.com/permissions and revokes the connected app's access.
  2. Run get_refresh_token.py again to generate a new token.
  3. Update it in the connector form.
Buenas prácticasBest practices Usa un cliente OAuth dedicado solo a este conector; autoriza únicamente los dos scopes mínimos; nunca compartas el client_secret ni el refresh_token por canales no seguros (chat, correo, capturas de pantalla); y revisa periódicamente en myaccount.google.com/permissions qué apps tienen acceso concedido al buzón. Use an OAuth client dedicated only to this connector; authorize only the two minimum scopes; never share the client_secret or refresh_token over insecure channels (chat, email, screenshots); and periodically review at myaccount.google.com/permissions which apps have granted access to the mailbox.
